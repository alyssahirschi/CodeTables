Semantic HTML

Element

Definition
<header> describes the content at the top of the page
<nav> encapsulates the page's navigational links
<main> encapsulates the main content between the header and footer areas
<footer> the page's footer content at the bottom of the body
<section> defines elements in a document ie chapters, headings, or any other area of the document with the same theme.
<article> holds the content that makes sense on its own usch as articles, blogs, and comments. Developers will generally use <section> to define a theme for the webpage and use <article> to write independent content for that theme.*article does not have to used with section.
<aside> is used to mark additional information that can enhance another element but isn't required in order to understand the main content.
